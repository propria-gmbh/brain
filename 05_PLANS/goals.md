# Цели

Цели привязаны к area-узлам в `tasks.json` через поле `goal_rank` (1/2/3), которое наследуется вниз по дереву задач (`parent_id`) — `tools/priority.py` использует это для расчёта G в формуле приоритета.

```
GOAL_RANK_SCORES = {1: 5, 2: 3, 3: 1}
```

## Зафиксированные (общие, не привязаны к area)

- **Здоровье** — физическое состояние, энергия, отсутствие боли и рисков
- **Душевное равновесие** — порядок в пространстве, радость от среды, в которой живу

## Высокоранговые area (не «цели» — это ранжирование проектов, не жизненные цели)

| Rank | Area | Что это |
|---|---|---|
| 1 | `area-safe-min` | Безопасный минимум — единственный высокоранговый (rank=1) сейчас |
| 2 | `area-health` | Здоровье |
| 2 | `area-ecom` | Ecom |
| 2 | `area-personal-family` | Семья |
| 3 | `area-ai-systems` | AI Системы |
| 3 | `area-personal-finance` | Personal Finance |

## Правило показа высокоранговых area

Каждый раз при обсуждении приоритетов — показывать все area с `goal_rank=1` (высокоранговые). Сейчас это только `area-safe-min`. Если в rank=1 несколько area — показывать все, без принудительного сокращения до 3.

## Изменить ранг

Менять `goal_rank` напрямую в tasks.json на нужном area-узле (требует подтверждения, см. `confirmation.md`).
