# Аудит системы — предложения

## 1. Автоматизация

### Сейчас нет, но можно сделать быстро

**A. Git auto-commit в конце дня**
- Скрипт или Claude cron: каждый вечер в 22:00 — `git add -A && git commit -m "daily snapshot $(date)"`
- Риск: коммитит незаконченные мысли. Решение: отдельная ветка `daily-log`, мержить вручную раз в неделю.

**B. Напоминания по дедлайнам**
- Сейчас дедлайны написаны в тексте задач (`дедлайн: 2026-05-14`), но ничто не напомнит.
- Решение: Claude cron — каждое утро читает все файлы задач, ищет `дедлайн:` с датой ≤ сегодня+2, присылает уведомление.

**C. Автогенерация today.md**
- Каждое утро: Claude берёт ждущие ответа задачи + просроченные дедлайны + перенесённые → формирует черновик today.md.
- Ты корректируешь, не создаёшь с нуля.

**D. Фиксация времени**
- VS Code extension может логировать активность (Wakatime или аналог).
- Альтернатива: простой скрипт, который пишет timestamp при открытии/закрытии brain/ файлов.

## 2. Правила — пробелы

**Нет правила архивирования выполненных задач**
- Сейчас выполненные задачи нигде не отмечаются — теряется история.
- Предложение: `- [x]` = выполнено, раз в неделю при Weekly Review — переносить в `06_DECISIONS/done-log.md`.

**Нет правила для "жду ответа"**
- Начали делать вручную в этой сессии. Надо оформить как правило.
- Предложение: раздел `## Ждёт ответа` в today.md — формат: `- кто/что (с YYYY-MM-DD, напомнить YYYY-MM-DD)`.

**Нет правила для проектов vs задач**
- Когда задача становится проектом (папка в `08_PROJECTS/`)?
- Предложение: проект = 3+ связанных задачи ИЛИ есть внешние участники ИЛИ есть деньги/обязательства.

**Нет правила для дублирования ecom.md и today.md**
- Сейчас задачи из ecom.md вручную копируются в today.md.
- Предложение: today.md ссылается на задачи (`→ ecom.md #поставщик`), не дублирует.

## 3. Учёт ивентов

**Проблема:** встречи/звонки (с конкретным временем) смешаны с задачами без времени.

**Предложение: добавить `05_PLANS/events.md`**

Формат:
```
## Предстоящие
- 2026-05-06 15:00 — физиотерапевт
- 2026-05-07 утро — тренировка (зал открывается)
- 2026-05-14 — дедлайн оплата Миши
- 2026-05-22 — картины на свадьбу

## Регулярные
- Пн-Сб 08:00-18:00 GMT+8 — KungFuBuy (рабочие часы для платежей)
- Еженедельно — Weekly Review
```

Это отдельно от задач — задачи не привязаны ко времени, события — привязаны.

## 4. Трекинг состояния (мигрень, сон, спорт, энергия)

**Предложение: daily state log в `04_THINKING/daily/YYYY-MM-DD.md`**

Шаблон (30 секунд на заполнение):
```yaml
---
date: 2026-05-06
energy: 4/10
sleep: плохой (мигрень)
workout: нет (зал закрыт)
migraine: да
mood: собранный к 11:00
---
```

**Зачем:** через месяц будет видно паттерны — мигрень после какого типа дней, когда энергия высокая, что предшествует хорошим рабочим дням.

**Автоматизация:** Claude cron утром создаёт файл с пустым шаблоном, ты заполняешь 30 секунд.
