## Стратегический вопрос — переразложен (коррекция)

**US store соло** раньше был неправильно свёрнут в «только performance для коучей». Пользователь уточнил: да, коучи — один из драйверов, **но если магазин заработает, деньги могут прийти быстрее чем с Митей — «очень вероятно»**.

**Коррекция:** US соло → **B (параллельный upside-трек, не C)**. Социальный драйвер отдельно — **письмо коучам с честным update** (20-30 мин) снимает стыд как отдельный слой, но не заменяет коммерческую гипотезу.

### Конфликт: два upside (Митя vs US) при незакрытом A1

**Правило:** A1 (baseline) не делим. Upside — максимум **один primary за раз**, второй — **parallel только если weekly capacity позволяет без краха A1**.

**Разрешение конфликта — 15-мин гипотеза-тест «time-to-first-dollar»**

Ответь честно на 4 вопроса (да/нет / число дней):

1. **Shopify US** — магазин уже существует и можно принять оплату сегодня/завтра? Если нет — сколько дней до этого?
2. **Каталог** — сколько SKU реалистично залить без listing_automation за 48-72ч? (если 0 — US не быстрый)
3. **Трафик** — откуда первые продажи: органика GMC / paid ads / email / другое? Есть ли уже работающий канал?
4. **GMC** — worst-case: сколько дней до approval если сейчас стартануть?

**Интерпретация:**
- Если суммарный best-case **time-to-revenue** US < best-case Митя (после его первого созвона) → **US становится primary upside на 1-2 недели**, Митя остаётся warm (не cold).
- Если US требует недели на инфра + 1000 SKU через pipeline → **Митя остаётся primary upside**, US — вторичный слот.

Этот тест — не meta. Это один лист бумаги / 15 мин в заметке. Результат фиксируется в `08_PROJECTS/ecom/stores/us-new/hypothesis-test.md`.

## Vaition — почему B, а не A

Честный разбор:
- Колоссальный upside, уже активно, есть задача (новая структура core)
- НО: нет revenue в 6 мес даже в оптимизме
- A2 уже занят upside-бетом (Митя)
- Два параллельных upside-бета при незакрытом baseline = расфокус

**Логика:** baseline (A1) → **primary upside (A2) — один**, остальные upside — parallel только если capacity позволяет → long-bet (Vaition) поддерживающим ритмом (B).

**Конкретное B-касание:** один **3-часовой deep-work слот на неделе** для «новой структуры core». Не ежедневные касания, не фон, не «всегда внимание». Один блок — и обратно к A1/A2.

**Примечание:** если US проходит тест быстрее Митя — **A2 может стать US**, Митя — warm. Решение после `hypothesis-test.md`, не до.
