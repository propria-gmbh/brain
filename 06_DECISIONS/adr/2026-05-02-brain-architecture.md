# ADR: brain/ architecture and role

## Context

Пользователь держит 13+ проектов в `~/Projects/`. Знания между сессиями теряются, решения импульсивные, контекст теряется при переключении, нет единого места для стратегии и рефлексии. Уже есть:
- Набор гайдов от Claude Code (`~/Projects/claude-setup/`, 23 layer-гайда)
- Готовая когнитивная структура в проекте `vaition` (00→99 pipeline)
- Cursor с глобальными скиллами (13 шт), но без глобальных rules
- SingularityApp для задач (Google Calendar 2-way sync, REST API), не интегрирован в мышление

Нужен «экзоскелет мышления», который:
1. Усиливает интеллект через принудительные протоколы (expert panel, adversarial review, loop interview)
2. Сохраняет знания в версионируемом формате
3. Соединяется с задачами и встречами
4. Работает из Cursor, с мобильного, голосом
5. Не размывает границу «планирование vs разработка»

## Alternatives considered

1. **Единый workspace с кодом + знаниями (brain + репо всё внутри).**
   - Плюсы: один рут, никаких cross-references.
   - Минусы: Claude Code проекты теряют изоляцию, brain/ распухает кодом, ADR и тикеты смешиваются с имплементацией, вся команда (если появится) вынуждена клонировать знания.

2. **Только Obsidian vault без git и без Cursor-интеграции.**
   - Плюсы: мобильно, просто, привычно.
   - Минусы: нет глобальных правил мышления в Cursor, нет автоматизации, git-версионирование только через плагин, интеграция с внешними проектами слабая.

3. **Notion / SaaS.**
   - Плюсы: полированный UX, встроенный мобильный.
   - Минусы: vendor lock-in, не markdown, не git, не работает с Cursor, выгрузка болезненная, конфиденциальность хуже.

4. **brain/ = командный центр, отдельно от Claude Code проектов; структура по vaition-паттерну; git + Obsidian + Cursor одновременно.** (Выбрано)

## Decision

Выбрали **вариант 4** — `~/Projects/brain/` как plain markdown + git + Obsidian vault + Cursor workspace одновременно, со структурой 00→99 по vaition-паттерну, реестром внешних Claude Code проектов в `08_PROJECTS/`, синхронизацией задач/встреч из SingularityApp и Google Calendar через скрипты в `tools/`.

Причины:
- **Разделение ответственностей**: brain/ = думать и помнить; Claude Code репо = делать и деплоить. Нельзя смешать без цены.
- **Переносимость**: plain markdown + git = ноль vendor lock-in. Obsidian — просто UI поверх, можно убрать.
- **Мобильность**: Obsidian Mobile + Working Copy работают с тем же vault.
- **Когнитивный pipeline**: vaition-структура уже доказана на практике пользователя, не надо изобретать.
- **Cursor integration**: `~/.cursor/rules/` даёт принудительные протоколы мышления глобально; `.cursor/rules/` в brain/ добавляет специфику.

## Consequences

**Легче:**
- Новая сессия в любом проекте: глобальные rules сразу дают стиль, безопасность, когнитивные протоколы.
- Знания не теряются при переключении — ADR, паттерны, сессии живут в одном git-реестре.
- Mobile/voice capture попадает в тот же vault.
- Видим общую картину по всем проектам через `08_PROJECTS/_index.md`.

**Сложнее:**
- Двойная бухгалтерия: часть решений описывается и в brain/, и в проектном репо. Требуется дисциплина cross-link.
- Скрипты синхронизации нужно написать и поддерживать (SingularityApp, Calendar).
- Obsidian + Cursor одновременно — нужно следить, чтобы конфиги не конфликтовали (`.obsidian/workspace*.json` в `.gitignore`).

**Обязаны теперь делать:**
- Каждый значимый выбор проходит через expert panel + adversarial review, оформляется ADR.
- Session close protocol — обязательная сводка в конце содержательной сессии.
- Еженедельный review с pattern-detector.
- Не класть prod-код в brain/.
- Не ставить систему на паузу: если 2 недели нет коммитов, что-то сломано в процессе.

## Review trigger

Пересмотреть:
- Через 3 месяца — проверить, действительно ли brain/ стал опорой, или превратился в свалку.
- Если 4+ недель нет новых ADR — паттерн «brain умирает», нужен ревайв.
- Если появится команда — пересмотреть модель shared vs personal brain.
