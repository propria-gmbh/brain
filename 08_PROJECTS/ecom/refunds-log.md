# Refunds log

Журнал **внешних** возвратов денег клиентам (partial / full). Источник истины по сумме и получателю — **Shopify order** + выписка платёжки; здесь — агрегат для brain и налоговой памяти.

## PII / git

**Зачем раньше писали «без имён»:** git хранит всё навсегда; при клонировании или утечке репозитория сочетание «ФИО + сумма возврата» — уже персональные данные. Email и платёжные реквизиты сюда **не** записываем вообще.

**Как можно у вас:** если репозиторий **строго частный**, удобно вести колонку **«Кому»** как имя с заказом (как в биллинге или в обращении) — так проще фильтровать «сколько и кому». Дубль по-прежнему сверять с Shopify. Если репозможен доступ к файлу посторонним — достаточно **инициалов / `#номер`** без ФИО.

## Конвенции колонок

| Колонка | Значение |
|---|---|
| Дата | дата фактического возврата (ISO `YYYY-MM-DD`) |
| Магазин | slug или бренд, например marc-francois |
| Store id | routing id витрины, напр. `qff1md-0e` |
| Заказ | `#1015` как в Shopify |
| Валюта | ISO 4217 |
| Сумма | число, точка как десятичный разделитель |
| Кому | имя как для себя или инициалы / `—`; email не нужен |
| Причина | delay, damage, withdrawal, goodwill, dup, другой код |
| Канал | Shopify refund, Stripe reversal, др. |
| Ref / txn | id транзакции в PSP или примечание из Shopify |
| Проверено | дата сверки с банком/Stripe |
