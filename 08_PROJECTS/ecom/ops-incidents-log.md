# Ops incidents log («косяки»)

Здесь фиксируем **внутренние и процессные сбои**: отгрузка без разрешения, длительная несостыковка со статусом, конфликт с поставщиком и т.п. — всё, из чего потом вырастает SOP или правило.

**Не путать с** жалобами клиента в Gorgias: там тикеты; здесь — **паттерны и факты по заказу** без лишнего PII. Идентификация — **номер заказа** + при необходимости Store id (`qff1md-0e` и др., см. `.cursor/rules/ecom-marc-francois-supplier-comms.mdc`).

## Типы (коды для колонки «Тип»)

| Код | Смысл |
|-----|--------|
| `ship_no_ok` | Отправили / отметили fulfilled без явного ок от нас на отправку |
| `supplier_delay` | Долго не отправляли / молчали по срокам |
| `carrier_damage` | Повреждение в пути |
| `data_mismatch` | Расходится статус: «у нас одно», «у них другое» |
| `other` | Вынести в описание |
