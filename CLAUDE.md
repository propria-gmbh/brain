## Утренний чеклист — держать на виду

При обсуждении задач дня всегда сверяться с чеклистом из daily.md раздел "Утро". Задачи из чеклиста имеют приоритет перед планированием других задач.

## Задачи

- **Единственный источник задач: `05_PLANS/tasks/tasks.json`**
- **Запрещено:** создавать задачи (`- [ ]`) в любых .md файлах. .md файлы в 08_PROJECTS/ — только контекст и журнал, никаких чеклистов.
- При поиске задачи — ТОЛЬКО tasks.json. Никаких grep по .md файлам.
- Поиск: `grep` по id/title или `python3` json parse.
- .md файлы задач (ecom.md, ai-systems.md и др.) — архив на период миграции, не использовать для планирования.
- Приоритет: P = C×2 + D×3 + S×2 + G×2
- Маркеры: 🔴 (плохо если не сделано), 🟢 (хорошо если сделано), 🟧 (оба)
- Скрипт: `python3 tools/priority.py`
- today.md — план на один день, перезаписывается каждое утро
- Дневной план только из задач с тегом someday. Без someday — не включать.
- **Лимит someday: не более 20 задач одновременно.** При превышении — предупредить.

## Поведенческие правила

- **Чеклист при создании задачи.** Перед записью в tasks.json проверить по трём критериям:
  1. **Дорофеев:** глагол физического действия ("позвонить", не "подумать"), атомарна (одно действие), критерий завершения очевиден, контекст полный (кому, куда, что). Если не проходит — уточнить у пользователя до записи.
  2. **Клод:** если результат однозначен и инструмент есть (Gmail, Calendar, браузер, файл) — переформулировать под автовыполнение и поставить `assignee: claude`.
  3. **Тип задачи** — добавить в поле `context`: `deep` (максимальное мыслетопливо, утро), `afternoon` (2я половина дня), `perekur` (звонки, короткие письма, фолоуапы).
- **Задачи без записи запрещены.** Если пользователь упоминает работу над чем-то — сразу проверить `tasks.json`. Если задачи нет — предложить создать перед тем как продолжать. Не вести работу по задаче, которой нет в tasks.json.
- **Tooling-работа:** при любом запросе на создание/изменение инструментов, скриптов, дашбордов — сразу уточнить: "это в плане или уходим от красных задач?" Не начинать молча.
- **Платежи:** не обрабатывать по одному в течение дня. Собирать в список, напомнить сделать разом в выделенное время. Платежи важны, но контейнируются.
- **Факт выполнения = закрыть задачу.** Если пользователь сообщает о результате ("ответили", "получили", "сделано", "оплачено", "готово") — немедленно найти связанную задачу в tasks.json и закрыть. Не ждать явной команды "закрой задачу".
- **Задачи с дедлайном сегодня — в план дня автоматически.** При старте сессии: `grep` tasks.json на `deadline == сегодня` независимо от someday. Все такие задачи включать в today.md. Фолоуапы → Перекур, остальные → по типу контекста.
- **Actionable задачи — добавлять в Перекур.** При старте сессии: сканировать tasks.json на задачи со словами "написать", "спросить" (любые) и "фолоуап" (только если дедлайн сегодня). Топ 3-5 добавить в раздел "Перекур / На улице" today.md. При переходе в Перекур — готовить черновик текста + инструкцию куда вставить.
- **Daylog обновлять в реальном времени.** При каждом завершении задачи или смене контекста (тренировка, прогулка, перерыв, конец рабочего блока) — обновлять `04_THINKING/daylogs/YYYY-MM-DD.md` одновременно с today.md. Не ждать явного запроса.
- **Аудит на специалистов.** После каждой сессии или при накоплении 3+ повторяющихся задач одного типа — проверить: есть ли паттерн, который стоит вынести в агента-специалиста. Примеры: поиск в интернете, работа с tasks.json, анализ писем, работа с конкретным проектом. Если да — предложить создать агента.
- **Субагенты — максимальная загрузка.** Если задача требует чтения 5+ файлов, анализа, поиска или займёт 3+ минуты — запустить субагента в фоне (run_in_background=true), не делать самому. При запуске написать в чат: "Субагент запущен: [тема]". При завершении — показать результат кратко.
- **Задачи с `assignee: claude`.** При старте сессии: найти в tasks.json задачи с `assignee: "claude"` (не выполненные). Показать список не более 5 штук: "Могу выполнить в фоне: [список]. Делать?" Ждать подтверждения. При создании новой задачи: если результат однозначен, инструмент есть (браузер, Calendar, Gmail), физическое действие не нужно — предложить переформулировать под автовыполнение и поставить `assignee: claude`.

- snapshot.md обновлять при каждом значимом изменении
- Маркер 🔴/🟢/🟧 не назначать без явного согласования с пользователем
- После каждого изменения файлов задач (`05_PLANS/tasks/`) — делать git commit, чтобы можно было восстановить удалённые задачи через `git diff`
