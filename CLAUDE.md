# brain — CLAUDE.md

Командный центр над проектами. Планирование, решения, контроль, синтез.

## Структура

```
00_META/       — snapshot.md (читать при старте сессии)
01_RULES/      — принципы планирования, snapshot-rule
02_CONTEXT/    — глоссарий, онтология
03_RESEARCH/   — внешние данные
04_THINKING/   — журнал сессий, гипотезы
05_PLANS/      — цели, задачи, today.md, recurring/
06_DECISIONS/  — ADR, записи встреч
07_OUTPUT/     — готовые артефакты
08_PROJECTS/   — реестр проектов (_index.md всегда актуален)
99_ARCHIVE/    — устаревшее
tools/         — priority.py и другие скрипты
```

## Старт каждой сессии

1. Прочитать `00_META/snapshot.md`
2. Прочитать `05_PLANS/today.md`
3. Прочитать `05_PLANS/recurring/daily.md`
4. **Первым сообщением пользователю** показать раздел "Утро" из daily.md — до планирования и задач
5. Проверить Gmail папку "1. Attention" через MCP
6. Проверить Gmail папку "0. Payments" через MCP — создать задачи по оплате и/или сохранению инвойсов
7. После фиксации задачи из письма в файлах — снять метку с письма

## Утренний чеклист — держать на виду

При обсуждении задач дня всегда сверяться с чеклистом из daily.md раздел "Утро". Задачи из чеклиста имеют приоритет перед планированием других задач.

## Задачи

- Файлы: `05_PLANS/tasks/*.md` и `05_PLANS/tasks/personal/*.md`
- Приоритет: P = C×2 + D×3 + S×2 + G×2
- Маркеры: 🔴 (плохо если не сделано), 🟢 (хорошо если сделано), 🟧 (оба)
- Скрипт: `python3 tools/priority.py`
- today.md — план на один день, перезаписывается каждое утро
- Дневной план только из задач с тегом [someday]. Без [someday] — не включать.

## Правила

- Имена задач — только дословно из файлов, никогда не перефразировать
- При создании/закрытии проекта в 08_PROJECTS/ — сразу обновить `08_PROJECTS/_index.md`
- snapshot.md обновлять при каждом значимом изменении
- Маркер 🔴/🟢/🟧 не назначать без явного согласования с пользователем
- После каждого изменения файлов задач (`05_PLANS/tasks/`) — делать git commit, чтобы можно было восстановить удалённые задачи через `git diff`
